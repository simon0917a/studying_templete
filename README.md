# 學習網站生成模板說明

這個資料夾是一套可重用的「靜態學習網站模板」，適合之後交給 AI 生成類似的教學網站。

目標網站形式：

- 1 個首頁：`index.html`
- 多個課程頁：例如 `course1.html`、`course2.html`
- 每個 HTML 都可獨立開啟，不依賴後端、不需要安裝套件
- 每個檔案都包含自己的 `<style>` 與 `<script>`
- 適合用來做心理學、SQL、科學、歷史、語言學習、通識課程等入門教學網站

## 模板檔案

- `home-template.html`：首頁模板  
  包含深藍漸層標題區、網站介紹、學習方式說明、課程卡片 grid、課程重點 badge、開始學習按鈕。

- `lesson-template.html`：課程頁模板  
  包含返回首頁按鈕、課程標題、導言、核心概念、生活例子、提示框、2 題單選互動測驗。

## 建議網站結構

```text
your-learning-site/
├── index.html
├── course1.html
├── course2.html
├── course3.html
├── course4.html
└── ...
```

## 教學風格要求

之後請 AI 生成學習網站時，建議明確要求以下教學風格：

1. 深入淺出  
   先用簡單語言定義概念，再逐步補充細節。不要一開始就堆疊艱深術語。

2. 生活化  
   每個抽象概念都要搭配具體生活例子，例如學校、工作、家庭、社群媒體、購物、遊戲、新聞、日常決策。

3. 結構清楚  
   每個課程頁至少包含：
   - 課程導言
   - 核心概念解釋
   - 生動的生活例子
   - 重點提示或記憶口訣
   - 2 題單選測驗
   - 每題的詳細答案解析

4. 像老師在講課  
   語氣要清楚、耐心、具體。不要只列點，也不要寫成百科全書。要讓初學者能跟著理解。

5. 概念拆解  
   遇到複雜概念時，要拆成小段落、小標題或條列說明。例如「定義」、「形成原因」、「常見誤解」、「生活應用」。

6. 測驗要有教學功能  
   測驗不是只顯示正確或錯誤。每個選項都要有 `data-explain`，說明為什麼對或錯，幫助學習者修正理解。

7. 內容要完整  
   不要用「略」、「此處省略」、「自行補充」等方式偷省內容。每個頁面都要能直接作為完整教材使用。

## UI 風格要求

這套模板的 UI 風格是簡潔、現代、教學導向：

- 首頁使用深藍漸層 header
- 主背景使用淡灰藍色
- 內容區塊使用白色卡片
- 卡片有圓角、細邊框與輕陰影
- 課程頁是單欄閱讀版面，適合長文教學
- `h1` 使用深藍文字與底線
- `h2` 使用左側藍色直線，方便掃讀
- 提示框使用淡藍背景與左側藍線
- 測驗區使用淡灰背景與白色選項按鈕
- 正確答案用綠色提示，錯誤答案用紅色提示
- 不使用複雜動畫，只保留 hover、按鈕回饋等簡單互動

## 可直接貼給 AI 的生成提示詞

你可以把下面這段直接貼給 AI，讓它依照這個模板生成新的學習網站。

```text
你是一位精通教學設計與前端開發的專家。請幫我製作一個純 HTML 的學習網站。

網站形式：
- 產生 1 個首頁 index.html
- 產生 N 個獨立課程頁 course1.html 至 courseN.html
- 每個 HTML 檔案都必須包含自己的 <style> 與 <script>
- 不使用外部框架、不使用後端、不需要安裝套件
- 所有頁面都要能直接用瀏覽器開啟

首頁要求：
- 使用深藍漸層 header
- 包含網站標題、簡短介紹、學習方式說明
- 使用 grid 排列課程卡片
- 每張卡片包含：
  - 課程編號
  - 課程標題
  - 內容簡介
  - 學習重點 badge
  - 「開始學習」按鈕
- 點擊卡片可前往對應課程頁

課程頁要求：
- 每頁頂部有「返回首頁」按鈕
- 使用單欄白色閱讀容器
- 課程內容要詳細、深入淺出、容易理解
- 每頁至少包含：
  - 課程導言
  - 核心概念解釋
  - 生動的生活例子
  - 提示框、總結或記憶口訣
  - 2 題單選測驗
- 測驗必須使用 JavaScript 即時回饋
- 點擊選項後顯示「正確」或「錯誤」
- 每個選項都要提供詳細答案解析，不只告訴答案

教學風格：
- 用初學者能懂的語言
- 先定義概念，再逐步拆解
- 每個抽象概念都要搭配生活例子
- 語氣像老師講解，不要像百科條目
- 避免空泛描述，要具體、有情境、有例子
- 不要省略內容，不要寫「略」或「自行補充」

UI 風格：
- 簡潔、現代、專業、好讀
- 主色使用深藍與亮藍
- 背景使用淡灰藍
- 卡片使用白底、圓角、細邊框、輕陰影
- 標題層級清楚，h1 有底線，h2 有左側藍線
- 測驗選項要像表單按鈕，有 hover 效果
- 不需要複雜動畫

請直接輸出完整 HTML 程式碼，不要省略任何文字。
```

## 使用方式

1. 複製 `home-template.html`，改名成 `index.html`。
2. 複製 `lesson-template.html`，依照課程數量改名，例如 `course1.html`、`course2.html`。
3. 修改首頁中的課程卡片：
   - `href`
   - 課程編號
   - 課程標題
   - 課程簡介
   - 學習重點 badge
4. 修改課程頁內容：
   - `<title>`
   - `COURSE 1`
   - `h1`
   - 導言
   - 核心概念
   - 生活例子
   - 小測驗題目與解析
5. 測驗正確答案請設定：

```html
<button class="option" data-correct="true" data-explain="正確解析文字">A. 正確選項</button>
```

錯誤答案請設定：

```html
<button class="option" data-correct="false" data-explain="錯誤原因解析">B. 錯誤選項</button>
```

## 課程頁內容建議格式

每個課程頁可依照以下格式撰寫：

```text
課程導言：
用一段話說明這一課要解決什麼問題。

核心概念解釋：
先定義主要概念，再拆成 2-4 個小重點。

生動的生活例子：
用具體情境說明概念如何出現在日常生活。

提示框：
放記憶口訣、常見誤解、學習提醒或總結。

課後小測驗：
兩題單選題，每題四個選項，每個選項都有解析。
```

## 注意事項

- 如果要生成多頁網站，請確保首頁連結和課程頁檔名一致。
- 如果複製課程頁模板，記得修改每頁的 `<title>` 和 `COURSE 編號`。
- 若內容是給初學者，請避免過多專業術語；必要術語要先解釋。
- 若用 AI 產生內容，請要求 AI「不要省略文本」，否則它可能只給摘要。
- 若要維持一致風格，請盡量不要在不同頁面使用完全不同的 CSS。
