# 學習網站生成模板說明

這個資料夾是一套可重用的「靜態學習網站模板」，適合之後交給 AI 生成類似的教學網站。

目標網站形式：

- 1 個首頁：`index.html`
- 多個課程頁：例如 `course1.html`、`course2.html`
- 每個 HTML 都可獨立開啟，不依賴後端、不需要安裝套件
- 每個檔案都包含自己的 `<style>` 與 `<script>`
- 適合用來做心理學、SQL、科學、歷史、語言學習、通識課程等入門教學網站

## 模板檔案

- `home-template.html`：首頁模板  
  包含深藍漸層標題區、網站介紹、學習方式說明、課程卡片 grid、課程重點 badge、開始學習按鈕。

- `lesson-template.html`：課程頁模板  
  包含返回首頁按鈕、課程標題、導言、核心概念、生活例子、提示框、2 題單選互動測驗。

## 建議網站結構

```text
your-learning-site/
├── index.html
├── course1.html
├── course2.html
├── course3.html
├── course4.html
└── ...
```

## 教學風格要求

之後請 AI 生成學習網站時，建議明確要求以下教學風格，並強調「深入探討」而非表面說明：

1. 先給定義，再揭開背後機制  
   先用一句話說清楚概念，接著立刻回答「為什麼會這樣？」「它是怎麼運作的？」  
   - 例如教「認知失調」，不只說定義，還要說明大腦如何尋求一致性、哪些實驗支持、日常如何觸發。

2. 多層次生活例子：從日常到進階  
   每個抽象概念至少提供兩種層次的生活例子：  
   - 表層例子（人人都碰得到，例如社群媒體的推薦機制）  
   - 深層例子（帶出矛盾、例外或情境變化，例如為何同一個機制在新聞和政治傳播上會失效）  
   例子要有前因後果，不要只是貼標籤。

3. 結構清楚且逐步加深  
   每個課程頁的內容必須形成「爬梯子」的知識結構：  
   - 課程導言：點出一個具體問題或迷思  
   - 核心概念解釋：定義 → 拆分要素 → 說明形成原因或運作流程  
   - 常見誤解與澄清：列出初學者最容易誤會的點  
   - 生活應用與延伸：不只舉例，還要分析「在什麼條件下會不同」  
   - 連結其他概念：點出這個知識和哪些心理學/科學/歷史概念有關

4. 老師在講課，更在引導思考  
   語氣清楚耐心，但要不時用提問引導：  
   - 「你有沒有想過，為什麼……？」  
   - 「如果情況變成……，結果還會一樣嗎？」  
   讓學習者感受到思考的推進，而非被動吸收。

5. 概念拆解要看到內部齒輪  
   複雜概念強制拆成：  
   - 定義  
   - 關鍵要素（或階段）  
   - 形成原因（心理/社會/生物等）  
   - 變異與例外  
   - 與其他概念的差異對比  
   禁止只寫「包含 A、B、C」而不解釋 A、B、C 的關聯。

6. 測驗要能檢驗深層理解  
   不只考定義記憶，每題至少兩個選項涉及常見誤解或概念混淆。  
   每個選項都要有 `data-explain`，解析要指出為什麼對、為什麼錯，並引導修正思維路徑。

7. 內容完整且拒絕表面敷衍  
   禁止使用「略」「此處省略」「讀者可自行查找」等偷懶寫法。  
   每一個段落都要能讓學習者讀完後，對該知識點有「啊，原來是這樣」的恍然大悟感。

## UI 風格要求

這套模板的 UI 風格是簡潔、現代、教學導向：

- 首頁使用深藍漸層 header
- 主背景使用淡灰藍色
- 內容區塊使用白色卡片
- 卡片有圓角、細邊框與輕陰影
- 課程頁是單欄閱讀版面，適合長文教學
- `h1` 使用深藍文字與底線
- `h2` 使用左側藍色直線，方便掃讀
- 提示框使用淡藍背景與左側藍線
- 測驗區使用淡灰背景與白色選項按鈕
- 正確答案用綠色提示，錯誤答案用紅色提示
- 不使用複雜動畫，只保留 hover、按鈕回饋等簡單互動

## 可直接貼給 AI 的生成提示詞

你可以把下面這段直接貼給 AI，讓它依照這個模板生成新的學習網站。

```text
你是一位精通教學設計與前端開發的專家。請幫我製作一個純 HTML 的學習網站。

網站形式：
- 產生 1 個首頁 index.html
- 產生 N 個獨立課程頁 course1.html 至 courseN.html
- 每個 HTML 檔案都必須包含自己的 <style> 與 <script>
- 不使用外部框架、不使用後端、不需要安裝套件
- 所有頁面都要能直接用瀏覽器開啟

首頁要求：
- 使用深藍漸層 header
- 包含網站標題、簡短介紹、學習方式說明
- 使用 grid 排列課程卡片
- 每張卡片包含：
  - 課程編號
  - 課程標題
  - 內容簡介
  - 學習重點 badge
  - 「開始學習」按鈕
- 點擊卡片可前往對應課程頁

課程頁要求：
- 每頁頂部有「返回首頁」按鈕
- 使用單欄白色閱讀容器
- 課程內容要詳細、深入淺出、容易理解
- 每頁至少包含：
  - 課程導言
  - 核心概念解釋
  - 生動的生活例子
  - 提示框、總結或記憶口訣
  - 2 題單選測驗
- 測驗必須使用 JavaScript 即時回饋
- 點擊選項後顯示「正確」或「錯誤」
- 每個選項都要提供詳細答案解析，不只告訴答案

教學風格：
- 用初學者能懂的語言
- 先定義概念，再逐步拆解
- 每個抽象概念都要搭配生活例子
- 語氣像老師講解，不要像百科條目
- 避免空泛描述，要具體、有情境、有例子
- 不要省略內容，不要寫「略」或「自行補充」

UI 風格：
- 簡潔、現代、專業、好讀
- 主色使用深藍與亮藍
- 背景使用淡灰藍
- 卡片使用白底、圓角、細邊框、輕陰影
- 標題層級清楚，h1 有底線，h2 有左側藍線
- 測驗選項要像表單按鈕，有 hover 效果
- 不需要複雜動畫

請直接輸出完整 HTML 程式碼，不要省略任何文字。
```

## 使用方式

1. 複製 `home-template.html`，改名成 `index.html`。
2. 複製 `lesson-template.html`，依照課程數量改名，例如 `course1.html`、`course2.html`。
3. 修改首頁中的課程卡片：
   - `href`
   - 課程編號
   - 課程標題
   - 課程簡介
   - 學習重點 badge
4. 修改課程頁內容：
   - `<title>`
   - `COURSE 1`
   - `h1`
   - 導言
   - 核心概念
   - 生活例子
   - 小測驗題目與解析
5. 測驗正確答案請設定：

```html
<button class="option" data-correct="true" data-explain="正確解析文字">A. 正確選項</button>
```

錯誤答案請設定：

```html
<button class="option" data-correct="false" data-explain="錯誤原因解析">B. 錯誤選項</button>
```

## 課程頁內容建議格式

每個課程頁請依照以下格式撰寫，確保不只是鋪陳，而是逐步挖深：

```text
課程導言（點燃好奇）：
- 用一個生活矛盾、常見迷思或有趣問題開場
- 明確告訴學習者：學完這一課，你就能解釋某個現象或避開某種錯誤

核心概念解釋（先定義，再拆解）：
- 定義：一句話說清概念
- 機制/原理：為什麼會發生？心理或運作機制是什麼？
- 關鍵要素拆解：列出 2-4 個構成要素，並說明彼此關係
- 舉一個「表面例子」說明基本運作
- 再舉一個「條件變化例子」說明何時會不同、為何不同

常見誤解與澄清：
- 列出初學者最容易搞混的 2-3 個點
- 逐一用正確觀念對比錯誤觀念，並說明錯誤從何而來

生活案例放大鏡（深入分析）：
- 選擇一個具體情境（如消費、學習、人際衝突、決策）
- 逐步拆解情境中如何出現該概念
- 分析背後的心理歷程或因果鏈
- 點出若缺乏此概念，人容易怎麼想、怎麼犯錯

延伸連結與思考（跨出單點）：
- 指出這個概念和哪個其他心理學/科學概念相關
- 提供一個「進階思考問題」，鼓勵學習者反思或觀察自己生活

提示框/記憶口訣（精華收束）：
- 放一句話總結或記憶口訣
- 可加入「何時該想起這個概念」的提醒

課後小測驗（檢測深層理解）：
- 兩題單選題，每題四個選項
- 至少兩題都要有「混淆選項」是來自常見誤解
- 每個選項必須有解析，錯誤選項的解析要指出具體的思考盲點
```

## 注意事項

- 如果要生成多頁網站，請確保首頁連結和課程頁檔名一致。
- 如果複製課程頁模板，記得修改每頁的 `<title>` 和 `COURSE 編號`。
- 若內容是給初學者，請避免過多專業術語；必要術語要先解釋。
- 若用 AI 產生內容，請要求 AI「不要省略文本」，否則它可能只給摘要。
- 若要維持一致風格，請盡量不要在不同頁面使用完全不同的 CSS。
